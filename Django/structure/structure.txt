application:
	admin.py - регистрируем модели для добавления их в систему администрирования Django 
	apps.py - файл который отвечает за настройки приложения (основная конфигурация приложения)
	migrations - содержит миграции баз данных приложения. Миграции позволяют Django отслеживать
		     изменения моделей и синхронизировать их со схемой бд
	models.py - обязательный файл. модели приложения
	tests.py - тесты для приложения
	views.py - обработчики. каждый обработчик получает HTTP-запрос, обрабатывает его и возвращает ответ
