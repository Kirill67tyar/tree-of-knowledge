Иногла в  проектах необходимо взаимодействие с другими системами,
для отображения данных нашего приложения.

API - специальный интерфейс, который определяет точки взаимодействия двух систем,
фронтенда, который хочет забрать у нас данные, и бэкенда, который эти 
данные предоставляет в JSON формате.

Есть множетво способ организации API, но самый распространённый - REST 

REST (от англ. Representational State Transfer — «передача репрезентативного состояния»

Такой способ взаимодействия двух систем основан на ресурсах. 

Данные в бд - ресурсы
HTTP-методы - действия

REST - возможность получать, создавать, редактировать, удалять
(в зависимости от дотупа конечно) ресурсы в нашей бд
через HTTP-запросы, на определённый URL с определёнными методами,
которые делает фронтенд.
Сам фронтенд ориентируется на коды ответов
