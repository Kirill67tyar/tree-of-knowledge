Рекомендации по открытию и увязке репозиториев в GITHUB и машине linux (ubuntu).
Надеюсь, что пост позволит избежать напрасной потери времени малоопытным пользователям и не считаю его подсказкой, т.к. к написанию кода задачи он не имеет отношения.  
1. Регистрируемся на GITHUB и создаем пустой репозиторий с любым именем КРОМЕ 'web'.
2. Инициируем репозиторий путем создания файла README. Инициализация файлом .gitignore может создать проблемы при операциях push. Предлагаемые на форуме шаблоны содержимого файла .gitignore у меня не пошли, получил в ответ:

"Следующие пути игнорируются одним из ваших файлов .gitignore:"

3. Очень ВАЖНЫЙ момент! При инициализации репозитория любым файлом мы тем самым открываем редактор COMMIT (создаем commit) на ветке main, которая является главной в репозитории на GITHUB. В нашем случае необходимо закрыть коммит, т.е. принять изменения (push). Непосредственно в репозитории сайта  такая возможность есть, я это делал при работе с уже существующими репозиториями, но как это сделать при открытии я не знаю, да это и не важно, мы это сделаем из командной строки домашнего компьютера (см ниже). 
4. Создаем на домашней машине папку проекта с любым именем КРОМЕ 'web'. Входим в нее и инициализируем пустой репозиторий git в этой папке:

git init

5. Клонируем в домашнюю папку репозиторий с сайта:  

git clone https://github.com/<ваш логин на GITHUB>/<имя репозитория на GITHUB>

6. В вашей домашней папке должна появиться папка с именем репозитория GITHUB. Вся дальнейшая работа по проекту проводится именно в этой папке!
7. Настройте внешний репозиторий с помощью:

git remote add <ваш логин на GITHUB> <полный адрес репозитория на GITHUB>

8. Вы уже можете создавать файлы и папки проекта, но команда git add даст следующую ошибку:

fatal: Не удалось создать «/home/user/.../.git/index.lock»: Файл существует.

Похоже, что другой процесс git запущен в этом репозитории,
например редактор открыт из «git commit». Пожалуйста, убедитесь,
что все процессы были завершены и потом попробуйте снова.

8. Иными словами пора закрыть редактор COMMIT! Для этого подтяните изменения с сайта:

git pull https://github.com/<ваш логин на GITHUB>/<имя репозитория на GITHUB> main

9. Теперь попробуйте закрыть commit:

sudo git push https://github.com/<ваш логин на GITHUB>/<имя репозитория на GITHUB>

Здесь необходим комментарий. Команда от имени админа или исполнится, или вы получите развернутый ответ с рекомендациями, поэтому, если команда не исполняется , попробуйте sudo ... Ну и пользуйтесь: git status.

10. Если ответ:

fatal: Текущая ветка master не имеет вышестоящей ветки.

11. Тогда:
git push --set-upstream https://github.com/<логин на GITHUB>/<имя репозитория на GITHUB> master

12. Убедитесь при помощи git status, что git кристально чист и вперед.
13. Не зря этот пункт 13! Если у вас ничего не получается и вы не понимаете состояние вашего git:
    - на GITHUB создайте новый пустой репозиторий и удалите старый;
    - на домашней машине обязательно удалите рабочую папку и создайте новую;
    - НА СЛЕДУЮЩИЙ ДЕНЬ начните с п. 2!
